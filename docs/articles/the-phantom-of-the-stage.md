# The Phantom of the Stage

*Staging in Locus — code that writes code at compile time, then vanishes.*

> *Part II of a diptych. In [Part I](the-phantom-of-the-handler.md) a handler did
> its work and left no trace. Here a whole program does — at the other stage.*

Locus has two halves. One is effects — *what code does* — and its ghost is the
handler that disappears once it's discharged. The other is **staging** — *when
code runs* — and it has a phantom of its own, with a better pun: it performs on
the generation **stage**, at compile time, and when the curtain falls the stage is
empty. Only the code it wrote remains.

## Two stages

A Locus program runs at **stage 0** — ordinary runtime. But you can write code
that runs *earlier*, at the **generation stage** (stage 1), whose job is to
produce stage-0 code. Two operators move between them:

- `quote(e)` builds a piece of code — a value of type `Code[T]` — without running
  it. It raises the stage.
- `${ c }` (splice) drops a generated piece of code back into the program. It
  lowers the stage, and it's where a generator *performs*: the splice runs its
  generator at compile time and leaves the residual code in its place.

The simplest trick is selection. This generator picks which code to emit based on
a condition known at the generation stage:

```
${ if 1 < 2 then quote(10) else quote(20) }
```

The `1 < 2` is decided during compilation, the `then` arm is chosen, and the
`else` arm *never exists in the program*. The assembly:

```asm
__locus_main:
    movl  $10, %eax
    retq
```

No comparison. No branch. Just the chosen constant. The generator ran and vanished
— exactly the way a discharged handler folds to a `mov`. Same ghost, other stage.

## A value that crosses the curtain

A generation-stage computation can produce a number — and that number can be
**baked into the generated code as a constant**. Watch a compile-time factor cross
from stage 1 into stage 0:

```
let scale = fn input: Int =>
  input * ${ let factor = 2 * 3 + 1 in quote(factor) }
in
scale 6
```

`factor` is computed during generation: `2 * 3 + 1 = 7`. `quote(factor)` lifts
that `7` across the stage boundary into the body of `scale`, which therefore *is*
`input * 7`. None of the `2 * 3 + 1` survives — the compiler did that arithmetic
and threw it away, leaving the constant it produced. The assembly multiplies by
seven and nothing else:

```asm
__locus_lam_0:                 ; scale's body
    leaq  (,%rdx,8), %rax      ; input * 8
    ...                        ;   - input  →  input * 7
```

(LLVM even strength-reduces `× 7` to `8x − x`.) A compile-time value, vanished into
a runtime constant.

## The aria: `power`

The famous staging example is `power` — and it is where the phantom truly sings.
`power n x` is a *recursive* code-builder: given a static exponent `n` and a piece
of code `x`, it generates `x * (x * ( … * 1))` with `n` factors. We define it
**inside** the splice, so the whole recursion lives at the generation stage:

```
let cube = fn y: Int =>
  ${ let rec power : Int -> Code[Int] -> Code[Int] =
       fn n: Int => fn x: Code[Int] =>
         if n == 0 then quote(1)
         else quote( ${x} * ${power (n - 1) x} )
     in power 3 (quote(y)) }
in
cube 4
```

`power 3 (quote y)` runs at compile time. It recurses three deep, splicing each
factor into the next, and produces the code `y * (y * (y * 1))`. So `cube` *is*
`fn y => y³`. The recursion, the `power` function, the `Code` values — every bit of
it is a generation-stage phantom. Here is what's left on the stage:

```asm
__locus_lam_0:                 ; cube's body
    imulq  %rdx, %rax
    imulq  %rdx, %rax
    retq
```

Two multiplies. (The `* 1` folded away; `y * (y * y)` is two `imul`s.) **No loop.
No recursion. No `power` anywhere in the program.** A recursive function ran to
completion at compile time and left three lines of arithmetic. `cube 4 = 64`,
`5³ = 125`, `2⁵ = 32` — each exponent specialized to straight-line multiplies,
because the builder performed and then vanished.

## When the phantom carries a tune

The two halves of Locus are supposed to *compose* — that's the whole bet, the
distributive law δ. And they do: a generator can build **effectful** code, and the
effect rides along inside the `Code`.

```
${ let level = 2 in
   if level == 2 then quote(writeln "[verbose] generated by a stage-1 program")
   else quote(writeln "[quiet]") }
```

`writeln` performs the `winapi` effect, so `quote(writeln …)` has type
`Code[Unit ! {winapi}]` — δ keeps the object effect *inside* the code. The
generator chooses the verbose variant at compile time; the program's type is
`Unit ! {winapi}`; and when the residual code runs, it prints. Staging decided
*which* code; the effect came with it; the row stayed honest about both. (The type
checker is even precise enough to refuse a staged `if` whose branches carry
*different* effects — `Code[Int ! {winapi}]` is not `Code[Int]` — because the
effect is part of the code's type.)

δ has a second direction, too. Where an *object* effect like `winapi` stays
**inside** the quote, a *generative* effect distributes **out** of it. The one
that matters in practice is `genlet` — let-insertion. A generator can ask for a
piece of code to be hoisted to a shared `let` and get back a reference, so a
subterm used many times is emitted once:

```
${ let r = genlet(quote(let u = writeln "[base]" in 5)) in
   quote(${r} + ${r}) }
```

For *pure* code this is invisible — LLVM's CSE already shares `y + 1`. But you
cannot dedup a side effect, so here `genlet` is the difference between the base
running **once** and **twice**: both evaluate to `5 + 5 = 10`, but only the print
count changes. The generative `Insert` effect — "I want a binding here" —
distributed out of the quote and became a `let` at the splice. Object effects
ride in the code; generative effects land at the locus; the distributive law keeps
them straight.

## The point

Staging is code that runs at compile time to produce the code that runs at
runtime. In Locus it is a real, recursive, Turing-complete computation — and like
its twin the handler, it is built to **disappear**:

- a static choice folds to the chosen constant — `mov $10`;
- a compile-time value is baked in as a literal — `× 7`;
- a recursive code-builder specializes and leaves only the arithmetic — two
  `imul`s, no `power`;
- and a quoted effect is carried, not dropped — the row composes.

Effects vanish when handled; staging vanishes when generated. They are the same
ghost wearing two masks — `A ! E` and `□A`, the graded monad and the graded
comonad — and the distributive law is the score that lets them sing together. You
write the opera. The machine plays an empty stage.

